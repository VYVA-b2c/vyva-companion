# VYVA Onboarding Module — Replit Agent Brief

## What this is
This is the complete frontend onboarding module for the VYVA B2C platform.
29 TypeScript/TSX files have been pre-built and are ready to use.
The setup script (vyva_setup.sh) creates all files automatically.

## Step 1 — Run the setup script
Upload vyva_setup.sh to this Replit project, then run in the Shell tab:
  bash vyva_setup.sh

This creates all 29 files in the correct folder structure under src/.

## Step 2 — Update shared/schema.ts
Replace shared/schema.ts with the approved schema v2.
Key additions: user_conditions, user_allergies, user_devices, trusted_providers,
subscription_plans, billing_events, stripe_webhooks tables.
Then run: npx drizzle-kit push

## Step 5 — Create server/routes/onboarding.ts
Full code already provided in the Replit build brief. Key routes:
  GET  /api/onboarding/state          — returns current onboarding state + features
  POST /api/onboarding/basics         — saves name, phone, language
  POST /api/onboarding/channel        — saves channel preferences
  POST /api/onboarding/consent        — records GDPR consent (required before activation)
  POST /api/onboarding/field          — marks a single field + unlocks features
  POST /api/onboarding/section/:id    — saves a complete profile section

## Step 6 — Create server/routes/billing.ts
Full code already provided. Key routes:
  GET  /api/billing/status            — returns plan, tier, trial days remaining
  POST /api/billing/create-checkout   — creates Stripe checkout session, returns URL
  GET  /api/billing/portal            — creates Stripe customer portal session
  POST /api/billing/webhook           — receives Stripe webhook events (needs raw body)

## Step 7 — Register routes in server/index.ts
  import onboardingRoutes from "./routes/onboarding";
  import billingRoutes from "./routes/billing";

  // Webhook MUST come before express.json() middleware
  app.post("/api/billing/webhook",
    express.raw({ type: "application/json" }),
    (req, res) => billingRoutes(req, res)
  );
  app.use(express.json());
  app.use("/api/onboarding", onboardingRoutes);
  app.use("/api/billing", billingRoutes);

## Step 9 — Verify end to end
Test this sequence:
  1. / → Welcome screen loads, path selection works
  2. Fill basics → POST /api/onboarding/basics succeeds → profile row in DB
  3. Channel step → consent screen → required toggle enables Continue button
  4. Activation screen → Complete my profile → Profile overview shows 0/11
  5. Open Medications → fill one med → save → unlock card appears → 1/11
  6. Open Care team → 4-step flow → consent toggles → invitation sent
  7. Settings → Privacy & consent → per-person toggles visible
  8. Settings → Subscription → Upgrade → Stripe checkout opens
